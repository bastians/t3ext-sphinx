.. _links-how-to-contribute:

Comment contribuer
^^^^^^^^^^^^^^^^^^

Cette extension utilise le même flux de travail que pour le noyau de TYPO3, mais se base sur les Pull Requests GitHub
pour le travail de relecture et d'approbation. Veuillez svp en lire plus à ce sujet dans le
`Guide de contribution TYPO3 <https://docs.typo3.org/typo3cms/ContributionWorkflowGuide/>`__ (anglais).


.. _links-how-to-contribute-rules:

Règles pour contribuer
""""""""""""""""""""""

- Il doit exister un ticket dans le bug tracker du projet expliquant le problème / l'amélioration proposée
- Les directives de codage `PSR-2`_ sont respectées
- Le message de commit respecte le `format utilisé par TYPO3`_ (la ligne "releases:" n'est pas nécessaire)
- Un seul changement logique par patch [#]_

.. _PSR-2: https://www.php-fig.org/psr/psr-2/
.. _format utilisé par TYPO3: https://docs.typo3.org/typo3cms/ContributionWorkflowGuide/Appendix/CommitMessage.html#commitmessage


.. rubric:: Notes de bas de page

.. [#] Le terme "patch" est utilisé dans le sense de "patch set" dans Gerrit, et peut correspondre à plusieurs
   commits successifs.
